My ONLINE PORTFOLIO

Context: Show my design and programming skills through my website portfolio


Duration: Ongoing


Collaboration: Individual


Tools: HTML, CSS, Git, Adobe Illustrator

I wanted to take the challenge as a UX designer and programmer to learn, create, and code my own portfolio to show off my design and programming skills. This is an ongoing project of designing, learning, improving, and coding.

GOALS



My main goal is to create and code this website portfolio to show my integrated programming and design skills. This strategy seemed like a fun, meta way to demonstrate my interests and skills.




I want my portfolio to be simple and modern while clearly communicating each project's development and process story. This includes creating strong organizational grids and maintaining consistency across all projects.




I am incoportating visuals and color because they are important design aspects. Visuals are able to tell the story of the design and its process while color engages the reader.

IDEATION


When thinking about my online portfolio, I looked at inspiration and knew that I wanted to keep it modern and simple while still telling the story of my projects.

I am interested in both design and computer science and want to to show both types of projects in my portfolio.

The three components of this website are the homepage, my projects (both programming and design), and contact me.





THE HOMEPAGE






On the home/about me page, I wanted to highlight two aspects of me: my education and work aspirations and fun facts about me.





My PROJECT PAGE(S)









On the design and programming pages I wanted to visually show each individual project I have completed by creating clickable squares with photos that represent each project. These photos are meant to interest the reader.

After clicking on a project, the read can learn more about that specific project. Across all projects, both design and programming, I wanted to state the project overview in a clear manner; this information is presented at the top of each project's page. After the overview, the project details are divided into smaller, digestable chunks of information that are accompanied by project photos.

I also wanted to share my key takeaways for each project, because all of my projects are important in my career, personal goals, and growth. For each project takeaway I intend to make an icon (on Illustrator) representing that takeaway.

CODING



I wanted to learn how to create a website. I started with a website template that mimicked what I wanted my website to look like and then altered the grid lines, photos, font sizes, and colors. I have self taught myself HTML and CSS through Googling and CodecAcademy and will be taking a web programming course at the UW in the near future. I still have a lot to learn (such as formatting my website for devices), but I realize my portfolio is an ongoing project that I will continuously improve.

TAKEAWAYS




Take bite size pieces

When I decided to create my own portfolio website I knew I had a lot to learn. For example, I had to learn a new coding language, design the layout to fit my vision, and write up the project stories. In order to efficiently accomplish this, I chunked the project into smaller tasks and spread them out over time. I see the value of this approach and will apply it to future projects to keep them manageable, organized, and on track.


Applying cements learning

Learning anything new can be challenging. I have a strong background in Java through robotics, programming courses, and internships, which helps me readily learn new languages. But I have learned that I grow and learn the most when I apply my new found knowledge to real life projects, like this portfolio.
